SQLite Manager

Docker 部署指南 v1.1.0

📦 镜像信息

属性值
镜像名称techfunways/sqlite-manage
版本v1.1.0 / latest
架构linux/amd64 + linux/arm64 多平台
基础镜像scratch（最小化，仅 ~12MB）
默认端口8080

🚀 快速开始

一行命令启动

docker run -d --name sqlite-manager -p 8080:8080 -v ./data:/data techfunways/sqlite-manage:v1.1.0

访问 http://localhost:8080 即可使用。

📁 目录结构

容器内/
├── app/
│   ├── sqlite-manager      # 可执行文件
│   └── www/                # 静态资源（只读）
│       ├── index.html
│       └── sqlite-web/
└── data/                   # 数据目录（需要挂载）
    ├── db/
    │   └── database.db     # 系统数据库
    ├── databases/          # 用户创建的数据库
    └── upload/             # 上传的文件
必须将 /data 目录挂载到宿主机，否则数据会丢失！

⚙️ 环境变量与参数

参数默认值说明
-port8080服务端口
-data-dir/data数据目录
-upload-dir/data/upload上传目录
-web-dir/app/www静态资源目录

🔧 常用命令

启动容器

docker run -d \
  --name sqlite-manager \
  -p 8080:8080 \
  -v ./data:/data \
  techfunways/sqlite-manage:v1.1.0

指定端口

docker run -d \
  --name sqlite-manager \
  -p 3000:3000 \
  -v ./data:/data \
  techfunways/sqlite-manage:v1.1.0 \
  -port 3000

查看日志

docker logs -f sqlite-manager

停止/启动

docker stop sqlite-manager
docker start sqlite-manager

重置密码

docker exec -it sqlite-manager \
  /app/sqlite-manager reset-password

查看版本

docker exec sqlite-manager \
  /app/sqlite-manager version

📋 Docker Compose

docker-compose.yml

version: '3.8'

services:
  sqlite-manager:
    image: techfunways/sqlite-manage:v1.1.0
    container_name: sqlite-manager
    ports:
      - "8080:8080"
    volumes:
      - ./data:/data
    restart: unless-stopped

启动：docker compose up -d

🔐 首次使用

访问应用

打开浏览器访问 http://localhost:8080

创建管理员账号

首次访问会提示创建管理员账号，输入用户名和密码

打开数据库

点击"浏览服务器文件"或"新建数据库"开始使用

💾 数据备份

备份

# 备份整个 data 目录
tar -czf sqlite-manager-backup-$(date +%Y%m%d).tar.gz ./data

# 或只备份数据库文件
cp ./data/db/database.db ./backup/database.db

恢复

# 停止容器
docker stop sqlite-manager

# 恢复数据
tar -xzf sqlite-manager-backup-20260401.tar.gz

# 启动容器
docker start sqlite-manager

🔄 更新镜像

拉取新版本

docker pull techfunways/sqlite-manage:latest

停止旧容器

docker stop sqlite-manager
docker rm sqlite-manager

启动新容器

docker run -d \
  --name sqlite-manager \
  -p 8080:8080 \
  -v ./data:/data \
  techfunways/sqlite-manage:latest
数据保存在 ./data 目录中，更新容器不会丢失数据。

🐛 故障排查

问题解决方案
页面 404检查 /data 目录是否正确挂载
无法创建数据库确保 ./data 目录有写权限：chmod 777 ./data
容器启动失败查看日志：docker logs sqlite-manager
端口冲突修改端口：-p 3000:8080
忘记密码运行：docker exec -it sqlite-manager /app/sqlite-manager reset-password

⚠️ 注意事项

首次运行必须创建管理员账号，否则无法登录使用。
建议定期备份 ./data 目录，防止数据丢失。
不要删除 ./data/db 目录，否则需要重新创建管理员账号。
