SQLite Manager

Docker 部署指南

📦 镜像信息

属性值
镜像名称techfunways/sqlite-manage
最新版本v1.0.0
支持架构linux/amd64, linux/arm64
基础镜像scratch（最小化）
默认端口8080

🚀 快速开始

拉取镜像

docker pull techfunways/sqlite-manage:v1.0.0

运行容器

docker run -d \
  --name sqlite-manager \
  -p 8080:8080 \
  -v ./data:/data \
  techfunways/sqlite-manage:v1.0.0

访问应用

打开浏览器访问 http://localhost:8080

📁 目录结构

容器内/
├── app/
│   ├── sqlite-manager      # 可执行文件
│   └── www/                # 静态资源（只读）
│       ├── index.html
│       └── sqlite-web/
└── data/                   # 数据目录（需要挂载）
    ├── db/
    │   └── database.db     # 系统数据库
    ├── databases/          # 用户创建的数据库
    └── upload/             # 上传的文件
建议将 /data 目录挂载到宿主机，确保数据持久化。

⚙️ 环境变量

变量默认值说明
SQLITE_DATA_DIR/data数据目录
SQLITE_UPLOAD_DIR/data/upload上传目录
SQLITE_WEB_DIR/app/www静态资源目录

🔧 常用命令

基本运行

docker run -d \
  -p 8080:8080 \
  -v ./data:/data \
  techfunways/sqlite-manage:v1.0.0

指定端口

docker run -d \
  -p 3000:3000 \
  -v ./data:/data \
  techfunways/sqlite-manage:v1.0.0 \
  -port 3000

查看日志

docker logs -f sqlite-manager

进入容器

docker exec -it sqlite-manager sh

重置密码

docker exec -it sqlite-manager \
  /app/sqlite-manager reset-password

停止/启动

docker stop sqlite-manager
docker start sqlite-manager

📋 Docker Compose

docker-compose.yml

version: '3.8'

services:
  sqlite-manager:
    image: techfunways/sqlite-manage:v1.0.0
    container_name: sqlite-manager
    ports:
      - "8080:8080"
    volumes:
      - ./data:/data
    restart: unless-stopped

运行：docker-compose up -d

🔐 首次使用

创建管理员账号

首次访问会提示创建管理员账号，输入用户名和密码即可。

打开数据库

点击"浏览服务器文件"选择数据库，或"新建数据库"创建新的数据库。

管理数据

点击左侧表名查看数据，双击单元格编辑数据。

⚠️ 注意事项

确保宿主机的 ./data 目录有读写权限，否则数据库无法正常工作。

权限设置

# 创建数据目录
mkdir -p ./data

# 设置权限（如果需要）
chmod 777 ./data

🔄 数据备份

备份数据目录

# 备份
tar -czf backup-$(date +%Y%m%d).tar.gz ./data

# 恢复
tar -xzf backup-20260401.tar.gz

🐛 故障排查

问题解决方案
页面 404检查是否正确挂载了 /data 目录
无法创建数据库确保 ./data 目录有写权限
容器启动失败查看日志：docker logs sqlite-manager
忘记密码运行：docker exec -it sqlite-manager /app/sqlite-manager reset-password
